Esta guía te lleva de cero a un pedido de pizza completado usando la Pizza API en solo cinco minutos. Autenticarás tu solicitud, buscarás en el menú un artículo para pedir, enviarás el pedido y confirmarás que fue aceptado.

## Requisitos previos

Antes de comenzar, asegúrate de tener:

- Una **clave de API** de Pizza API. Si aún no tienes una, consulta [Autenticación](/es/authentication) para generar una.
- Una herramienta para hacer solicitudes HTTP, como `curl`, o cualquier lenguaje con un cliente HTTP.
- La URL base de la API a mano: `https://api.pizzaapi.com/v1`.

## 1. Autentica tu solicitud

Cada llamada a la Pizza API debe incluir tu clave de API en la cabecera `Authorization` como token Bearer:

```bash
curl https://api.pizzaapi.com/v1/menu \
  -H "Authorization: Bearer YOUR_API_KEY"
```

Reemplaza `YOUR_API_KEY` con la clave que generaste. Si la clave es válida, recibirás una respuesta `200 OK` con el menú completo.

## 2. Explora el menú

El endpoint `/menu` devuelve todas las pizzas, acompañamientos y bebidas disponibles. Puedes filtrar por categoría para reducir los resultados:

```bash
curl https://api.pizzaapi.com/v1/menu?category=pizza \
  -H "Authorization: Bearer YOUR_API_KEY"
```

Cada elemento del menú incluye un `id`, un `name`, los `available_sizes` y `available_toppings`. Toma nota del `id` de la pizza que quieres pedir — lo usarás en el siguiente paso.

## 3. Realiza tu primer pedido

Envía una solicitud `POST` a `/orders` con el ID del artículo del menú, tamaño, cantidad e ingredientes:

<CodeGroup>

```bash cURL
curl -X POST https://api.pizzaapi.com/v1/orders \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "items": [
      {
        "menu_item_id": "item_margherita",
        "size": "medium",
        "quantity": 1,
        "toppings": ["extra_cheese"]
      }
    ],
    "delivery_address": "123 Main St, Springfield, IL 62701"
  }'
```

```javascript JavaScript
const response = await fetch("https://api.pizzaapi.com/v1/orders", {
  method: "POST",
  headers: {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    items: [
      {
        menu_item_id: "item_margherita",
        size: "medium",
        quantity: 1,
        toppings: ["extra_cheese"],
      },
    ],
    delivery_address: "123 Main St, Springfield, IL 62701",
  }),
});

const order = await response.json();
console.log(order);
```

</CodeGroup>

Una respuesta exitosa devuelve un objeto de pedido con un identificador único y un estado inicial `pending`:

```json
{
  "id": "ord_abc123",
  "status": "pending",
  "total": 13.99,
  "estimated_delivery_time": "2024-01-15T18:45:00Z"
}
```

## 4. Confirma tu pedido

Usa el `id` del pedido devuelto para verificar su estado:

```bash
curl https://api.pizzaapi.com/v1/orders/ord_abc123 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

El `status` cambiará a lo largo del ciclo de vida del pedido: `pending` → `confirmed` → `preparing` → `out_for_delivery` → `delivered`.

<Tip>
  En lugar de sondear el endpoint de pedidos, suscríbete a [webhooks](/es/guides/webhooks) para recibir notificaciones instantáneas cada vez que cambie el estado.
</Tip>
