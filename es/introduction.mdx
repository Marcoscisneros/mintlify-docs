Pizza API ofrece a los desarrolladores todo lo necesario para integrar un flujo completo de pedido de pizza en cualquier aplicación. Ya sea que estés construyendo una tienda para consumidores, un portal de almuerzo corporativo o un bot de pedidos automatizado, Pizza API gestiona el menú, los pedidos, el seguimiento de la entrega y los datos del cliente para que puedas centrarte en la experiencia.

## Características principales

Pizza API está diseñada para cubrir cada etapa del ciclo de pedido — desde explorar el menú hasta la entrega en la puerta — con una interfaz REST limpia y predecible.

<CardGroup cols={3}>
  <Card title="Explorar el menú" icon="list">
    Obtén el menú completo o filtra por categoría. Cada artículo incluye tamaños disponibles, ingredientes y precios.
  </Card>

  <Card title="Pedidos personalizados" icon="sliders">
    Combina ingredientes, tipos de masa y tamaños para construir cualquier pedido que tus clientes puedan imaginar.
  </Card>

  <Card title="Seguimiento de entrega" icon="truck">
    Consulta o suscríbete para recibir actualizaciones en tiempo real del estado del pedido, ubicación del conductor y hora estimada de llegada.
  </Card>

  <Card title="Perfiles de clientes" icon="user">
    Guarda direcciones de entrega y recupera el historial de pedidos para que los clientes recurrentes finalicen la compra más rápido.
  </Card>

  <Card title="Webhooks" icon="webhook">
    Recibe callbacks HTTP instantáneos cada vez que cambia el estado de un pedido, sin necesidad de sondeo.
  </Card>

  <Card title="Autenticación sencilla" icon="shield-halved">
    Basta con una clave de API enviada como token Bearer en la cabecera `Authorization`.
  </Card>
</CardGroup>

## URL base

Todos los endpoints de Pizza API se sirven desde la siguiente URL base:

```text
https://api.pizzaapi.com/v1
```

<Note>
  Cada solicitud debe realizarse sobre **HTTPS** y debe incluir una clave de API válida en la cabecera `Authorization`. Las solicitudes por HTTP simple o sin clave serán rechazadas con una respuesta `401 Unauthorized`.
</Note>
