Esta guía te lleva por el ciclo completo de realizar un pedido de pizza con Pizza API. Crearás o recuperarás un cliente, explorarás el menú, construirás tu pedido, lo enviarás y verificarás la respuesta — todo lo necesario para poner una pizza en camino.

<Steps>
  <Step title="Crea o recupera tu cliente">
    Antes de hacer un pedido necesitas un registro de cliente. Si es nuevo, créalo con `POST /customers`. Si ya existe, obténlo con `GET /customers/{id}`.

    ```json Cuerpo de la solicitud — POST /customers
    {
      "name": "Jamie Lee",
      "email": "jamie@example.com",
      "phone": "+1-555-0142",
      "address": "42 Maple Street, Springfield, IL 62701"
    }
    ```

    Una respuesta exitosa devuelve un `customer_id` que usarás al enviar el pedido:

    ```json Respuesta
    {
      "customer_id": "cus_abc123",
      "name": "Jamie Lee",
      "email": "jamie@example.com",
      "phone": "+1-555-0142",
      "address": "42 Maple Street, Springfield, IL 62701",
      "created_at": "2024-01-15T18:00:00Z"
    }
    ```
  </Step>
  <Step title="Explora el menú">
    Obtén las pizzas disponibles con `GET /menu` usando parámetros de consulta para filtrar por categoría y disponibilidad.

    ```text
    GET /menu?category=pizza&available=true
    ```

    La respuesta contiene una lista de artículos del menú con sus IDs, precios base y opciones de personalización:

    ```json Respuesta (abreviada)
    {
      "items": [
        {
          "id": "pizza_margherita",
          "name": "Margherita",
          "description": "Salsa de tomate clásica, mozzarella fresca y albahaca.",
          "base_price": 11.99,
          "available_sizes": ["small", "medium", "large"],
          "available_toppings": ["extra_cheese", "mushrooms", "bell_peppers", "olives"]
        },
        {
          "id": "pizza_pepperoni",
          "name": "Pepperoni",
          "description": "Salsa de tomate, mozzarella y pepperoni clásico.",
          "base_price": 13.99,
          "available_sizes": ["small", "medium", "large"],
          "available_toppings": ["extra_cheese", "jalapeños", "mushrooms", "onions"]
        }
      ]
    }
    ```

    Guarda el `id` de cada pizza que quiera tu cliente — lo necesitarás para construir el pedido.
  </Step>
  <Step title="Construye los artículos del pedido">
    Arma el array `items` que incluirás en la solicitud. Cada artículo referencia un artículo del menú por su ID y especifica tamaño, cantidad, ingredientes e instrucciones especiales.

    ```json array items
    [
      {
        "menu_item_id": "pizza_pepperoni",
        "size": "large",
        "quantity": 1,
        "toppings": ["extra_cheese", "mushrooms"],
        "special_instructions": "Masa bien cocida por favor"
      }
    ]
    ```

    Consulta la guía [Personalizar pizza](/es/guides/customize-pizza) para conocer todas las opciones de ingredientes y tamaños.
  </Step>
  <Step title="Envía el pedido">
    Publica el pedido completo en `POST /orders`, incluyendo el `customer_id`, tu array `items` y la dirección de entrega.

    <CodeGroup>

    ```bash curl
    curl --request POST \
      --url https://api.pizzaapi.com/v1/orders \
      --header 'Authorization: Bearer YOUR_API_KEY' \
      --header 'Content-Type: application/json' \
      --data '{
        "customer_id": "cus_abc123",
        "items": [
          {
            "menu_item_id": "pizza_pepperoni",
            "size": "large",
            "quantity": 1,
            "toppings": ["extra_cheese", "mushrooms"],
            "special_instructions": "Masa bien cocida por favor"
          }
        ],
        "delivery_address": "42 Maple Street, Springfield, IL 62701"
      }'
    ```

    ```javascript JavaScript (fetch)
    const response = await fetch("https://api.pizzaapi.com/v1/orders", {
      method: "POST",
      headers: {
        "Authorization": "Bearer YOUR_API_KEY",
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        customer_id: "cus_abc123",
        items: [
          {
            menu_item_id: "pizza_pepperoni",
            size: "large",
            quantity: 1,
            toppings: ["extra_cheese", "mushrooms"],
            special_instructions: "Masa bien cocida por favor",
          },
        ],
        delivery_address: "42 Maple Street, Springfield, IL 62701",
      }),
    });
    
    const order = await response.json();
    console.log(order);
    ```

    </CodeGroup>

    Una respuesta exitosa `201 Created` confirma que el pedido fue recibido:

    ```json Respuesta
    {
      "id": "ord_xyz789",
      "status": "pending",
      "customer_id": "cus_abc123",
      "total": 16.99,
      "estimated_delivery_time": "2024-01-15T19:45:00Z",
      "created_at": "2024-01-15T19:05:00Z"
    }
    ```
  </Step>
  <Step title="Maneja la respuesta">
    Tras enviar, revisa el campo `status`. Un pedido recién enviado tendrá el estado `pending`, es decir, recibido y a la espera de la aceptación de la cocina.

    Guarda el `id` devuelto — este es tu `order_id` y será necesario para todas las llamadas de seguimiento y gestión.

    ```javascript Guardar el ID del pedido
    const { id: orderId, status, estimated_delivery_time } = order;
    
    console.log(`El pedido ${orderId} está ${status}.`);
    console.log(`Entrega estimada: ${estimated_delivery_time}`);
    ```

    <Tip>
      Suscríbete a [webhooks](/es/guides/webhooks) para recibir notificaciones push instantáneas cuando cambie el estado del pedido. Es mucho más eficiente que sondear la API.
    </Tip>
  </Step>
</Steps>

<Note>
  Los pedidos pasan de `pending` a `confirmed` cuando la cocina los acepta. Si necesitas modificar el pedido — añadir artículos o cambiar la dirección — debes hacerlo **antes** de que el pedido llegue a `confirmed`. Después de eso, ya no se aceptan modificaciones.
</Note>
