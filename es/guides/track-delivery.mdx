Los endpoints de entrega de Pizza API te permiten construir experiencias de seguimiento en tiempo real — muestra a los clientes dónde está su pedido, quién lo entrega y cuándo llegará. Esta guía cubre cómo obtener el estado en vivo, recuperar estimaciones antes del despacho, entender el ciclo de estados y elegir la mejor estrategia para mantener tu UI al día.

## Obtener el estado de la entrega

Una vez despachado el pedido, llama a `GET /delivery/{order_id}/status` para obtener el estado actual, los datos del repartidor y la hora estimada de llegada.

<CodeGroup>

```bash curl
curl --request GET \
  --url https://api.pizzaapi.com/v1/delivery/ord_xyz789/status \
  --header 'Authorization: Bearer YOUR_API_KEY'
```

```javascript JavaScript (fetch)
const response = await fetch(
  "https://api.pizzaapi.com/v1/delivery/ord_xyz789/status",
  {
    headers: {
      Authorization: "Bearer YOUR_API_KEY",
    },
  }
);

const status = await response.json();
console.log(status);
```

</CodeGroup>

```json Respuesta
{
  "order_id": "ord_xyz789",
  "status": "in_transit",
  "driver": {
    "name": "Marco R.",
    "phone": "+1-555-0199",
    "vehicle": "Honda Civic rojo"
  },
  "estimated_delivery_at": "2024-01-15T19:45:00Z",
  "delivered_at": null
}
```

El campo `delivered_at` es `null` hasta que el repartidor marca el pedido como entregado, momento en que se establece la marca de tiempo de la entrega.

## Estimación antes del despacho

Incluso antes de asignar un repartidor, puedes mostrar una ETA al cliente llamando a `GET /delivery/{order_id}/estimate`. Este endpoint está disponible en cuanto el pedido alcanza el estado `confirmed`.

```text
GET /delivery/ord_xyz789/estimate
```

```json Respuesta
{
  "order_id": "ord_xyz789",
  "estimated_minutes": 35,
  "estimated_delivery_at": "2024-01-15T19:45:00Z"
}
```

Úsalo para fijar expectativas justo después de la confirmación del pedido, antes de que haya datos del repartidor.

## Ciclo de estados

Un pedido pasa por los siguientes estados desde la colocación hasta la entrega:

| Estado | Significado | UI sugerida |
| --- | --- | --- |
| `pending` | Esperando asignación de repartidor | "Pedido confirmado" |
| `assigned` | Repartidor camino a la cocina | "Preparando tu pedido" |
| `picked_up` | El repartidor tiene la pizza | "¡En camino!" |
| `in_transit` | En ruta hacia el cliente | "Llega en X min" |
| `delivered` | Entregado con éxito | "¡Disfruta tu pizza!" |

## Sondeo vs webhooks

### Sondeo

Para implementaciones simples o de bajo tráfico, puedes sondear el endpoint de estado a intervalos regulares:

```javascript Sondeo cada 30 segundos
const ORDER_ID = "ord_xyz789";
const POLL_INTERVAL_MS = 30_000;

const pollDeliveryStatus = async () => {
  const response = await fetch(
    `https://api.pizzaapi.com/v1/delivery/${ORDER_ID}/status`,
    {
      headers: { Authorization: "Bearer YOUR_API_KEY" },
    }
  );

  const data = await response.json();
  console.log("Estado actual:", data.status);

  if (data.status === "delivered") {
    console.log("¡Pedido entregado! Deteniendo el sondeo.");
    clearInterval(intervalId);
  }
};

const intervalId = setInterval(pollDeliveryStatus, POLL_INTERVAL_MS);
pollDeliveryStatus(); // ejecutar de inmediato
```

<Note>
  El endpoint de estado tiene un límite de **1 solicitud cada 30 segundos por pedido**. Sondear con mayor frecuencia devuelve `429 Too Many Requests`. Usa un intervalo de al menos 30 segundos.
</Note>

### Webhooks (recomendado para producción)

En producción usa [webhooks](/es/guides/webhooks) en lugar de sondear. Los webhooks empujan una notificación a tu servidor en el momento en que cambia el estado, eliminando solicitudes innecesarias y ofreciendo actualizaciones más rápidas sin carga extra.

Consulta la [guía de Webhooks](/es/guides/webhooks) para instrucciones completas, esquemas y verificación de firmas.
