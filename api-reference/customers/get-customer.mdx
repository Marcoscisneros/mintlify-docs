Use this endpoint to retrieve a complete customer profile by its unique ID. The response includes the customer's contact details, all saved delivery addresses, and the currently configured default address — useful for pre-filling order forms or displaying account information in your application.

## Endpoint

```
GET https://api.pizzaapi.com/v1/customers/{id}
```

## Path Parameters

<ParamField path="id" type="string" required>
  The unique customer ID returned when the profile was created (e.g. `cus_a1b2c3d4`).
</ParamField>

## Example Request

<CodeGroup>

```bash cURL
curl --request GET \
  --url https://api.pizzaapi.com/v1/customers/cus_a1b2c3d4 \
  --header 'Authorization: Bearer YOUR_API_KEY'
```

```javascript JavaScript
const customerId = 'cus_a1b2c3d4';

const response = await fetch(
  `https://api.pizzaapi.com/v1/customers/${customerId}`,
  {
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY',
    },
  }
);

const customer = await response.json();
console.log(customer.email); // "jamie.hernandez@example.com"
```

```python Python
import requests

customer_id = 'cus_a1b2c3d4'

response = requests.get(
    f"https://api.pizzaapi.com/v1/customers/{customer_id}",
    headers={"Authorization": "Bearer YOUR_API_KEY"},
)

customer = response.json()
print(customer["email"])  # "jamie.hernandez@example.com"
```

</CodeGroup>

### Response Fields

<ResponseField name="id" type="string">
  The unique identifier for the customer.
</ResponseField>

<ResponseField name="name" type="string">
  The customer's full name.
</ResponseField>

<ResponseField name="email" type="string">
  The customer's registered email address.
</ResponseField>

<ResponseField name="phone" type="string">
  The customer's phone number in E.164 format. `null` if not set.
</ResponseField>

<ResponseField name="addresses" type="array">
  All delivery addresses saved to this customer profile.

  <Expandable title="address object">
    <ResponseField name="addresses[].id" type="string">
      The unique identifier for this address.
    </ResponseField>

    <ResponseField name="addresses[].label" type="string">
      The friendly label for the address. `null` if not set.
    </ResponseField>

    <ResponseField name="addresses[].street" type="string">
      The full street address.
    </ResponseField>

    <ResponseField name="addresses[].city" type="string">
      The city name.
    </ResponseField>

    <ResponseField name="addresses[].state" type="string">
      The two-letter state abbreviation.
    </ResponseField>

    <ResponseField name="addresses[].zip" type="string">
      The ZIP or postal code.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="default_address_id" type="string">
  The `id` of the customer's preferred delivery address. `null` if no addresses have been saved.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp indicating when the customer profile was created.
</ResponseField>

<Note>
  If no customer exists for the provided `id`, the API returns a `404 Not Found` response. Ensure you are using the exact `id` value returned at customer creation — IDs are case-sensitive.
</Note>
