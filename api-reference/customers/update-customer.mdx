---
title: "PATCH /customers/{id} — Update Customer Details"
sidebarTitle: "Update Customer"
description: "Use PATCH /customers/{id} to update a customer's name, phone, default address, or saved addresses. Send only the fields you want to change."
---

This endpoint performs a partial update on an existing customer profile. You only need to include the fields you want to modify — any fields omitted from the request body will remain unchanged. This is useful for updating contact details, adding a new phone number, or changing the customer's preferred delivery address.

## Endpoint

```
PATCH https://api.pizzaapi.com/v1/customers/{id}
```

## Path Parameters

<ParamField path="id" type="string" required>
  The unique customer ID of the profile to update (e.g. `cus_a1b2c3d4`).
</ParamField>

## Request Body

All fields are optional. Include only the fields you wish to update.

<ParamField body="name" type="string">
  The customer's updated full name.
</ParamField>

<ParamField body="phone" type="string">
  The customer's updated phone number in [E.164 format](https://en.wikipedia.org/wiki/E.164) (e.g. `+15550009999`).
</ParamField>

<ParamField body="default_address_id" type="string">
  The `id` of an existing saved address to set as the customer's preferred delivery address. Must reference a valid address in the customer's `addresses` array.
</ParamField>

<ParamField body="addresses" type="array">
  A replacement list of delivery addresses. Sending this field **replaces the entire existing addresses array** — it is not a merge. Include all addresses you want to retain, plus any new ones.

  <Expandable title="address object">
    <ParamField body="addresses[].label" type="string">
      A friendly label for the address (e.g. `"Home"`, `"Work"`).
    </ParamField>

    <ParamField body="addresses[].street" type="string" required>
      The full street address including house or apartment number.
    </ParamField>

    <ParamField body="addresses[].city" type="string" required>
      The city name.
    </ParamField>

    <ParamField body="addresses[].state" type="string" required>
      The two-letter state abbreviation.
    </ParamField>

    <ParamField body="addresses[].zip" type="string" required>
      The ZIP or postal code.
    </ParamField>
  </Expandable>
</ParamField>

## Example Request

<CodeGroup>

```bash cURL
curl --request PATCH \
  --url https://api.pizzaapi.com/v1/customers/cus_a1b2c3d4 \
  --header 'Authorization: Bearer YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "phone": "+15550009999",
    "default_address_id": "adr_m3n4o5p6"
  }'
```

```javascript JavaScript
const customerId = 'cus_a1b2c3d4';

const response = await fetch(
  `https://api.pizzaapi.com/v1/customers/${customerId}`,
  {
    method: 'PATCH',
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      phone: '+15550009999',
      default_address_id: 'adr_m3n4o5p6',
    }),
  }
);

const updated = await response.json();
console.log(updated.default_address_id); // "adr_m3n4o5p6"
```

```python Python
import requests

customer_id = 'cus_a1b2c3d4'

response = requests.patch(
    f"https://api.pizzaapi.com/v1/customers/{customer_id}",
    headers={"Authorization": "Bearer YOUR_API_KEY"},
    json={
        "phone": "+15550009999",
        "default_address_id": "adr_m3n4o5p6",
    },
)

updated = response.json()
print(updated["default_address_id"])  # "adr_m3n4o5p6"
```

</CodeGroup>

### Response Fields

<ResponseField name="id" type="string">
  The unique identifier for the customer.
</ResponseField>

<ResponseField name="name" type="string">
  The customer's full name.
</ResponseField>

<ResponseField name="email" type="string">
  The customer's registered email address. Email cannot be changed via this endpoint.
</ResponseField>

<ResponseField name="phone" type="string">
  The customer's current phone number in E.164 format. `null` if not set.
</ResponseField>

<ResponseField name="addresses" type="array">
  The current list of delivery addresses after the update.

  <Expandable title="address object">
    <ResponseField name="addresses[].id" type="string">
      The unique identifier for this address.
    </ResponseField>

    <ResponseField name="addresses[].label" type="string">
      The friendly label for the address. `null` if not set.
    </ResponseField>

    <ResponseField name="addresses[].street" type="string">
      The full street address.
    </ResponseField>

    <ResponseField name="addresses[].city" type="string">
      The city name.
    </ResponseField>

    <ResponseField name="addresses[].state" type="string">
      The two-letter state abbreviation.
    </ResponseField>

    <ResponseField name="addresses[].zip" type="string">
      The ZIP or postal code.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="default_address_id" type="string">
  The `id` of the customer's preferred delivery address after the update.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the customer profile was originally created. This value never changes.
</ResponseField>

<Tip>
  To add a single new address without removing existing ones, first fetch the current customer profile via `GET /customers/{id}` to retrieve the full `addresses` array. Then send the complete array — existing entries plus the new one — in your `PATCH` request body. This ensures no saved addresses are accidentally removed.
</Tip>
