Modify an existing order's line items or delivery address. This endpoint uses partial update semantics — only include the fields you want to change. Providing `items` replaces the entire items array, and providing `delivery_address` replaces the entire delivery address.

```
PATCH https://api.pizzaapi.com/v1/orders/{id}
```

<Warning>
  This endpoint can only be called while the order has a `pending` status. Once the kitchen confirms the order, no further modifications are permitted and the API will return a `409 Conflict` error.
</Warning>

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the order to update (e.g. `order_55ABCD112233`).
</ParamField>

## Request Body

All fields are optional. Send only the fields you wish to update.

<ParamField body="items" type="array">
  Replaces the **entire** items array on the order. Must contain at least one item if provided. Follows the same structure as the `items` field in [Create Order](/api-reference/orders/create-order).

  <Expandable title="Item object fields">
    <ParamField body="items[].menu_item_id" type="string" required>
      The ID of the menu item.
    </ParamField>

    <ParamField body="items[].size" type="string" required>
      Desired size: `small`, `medium`, or `large`.
    </ParamField>

    <ParamField body="items[].quantity" type="integer" required>
      Number of this item. Minimum: `1`.
    </ParamField>

    <ParamField body="items[].toppings" type="array">
      Array of topping IDs for this item.
    </ParamField>

    <ParamField body="items[].special_instructions" type="string">
      Free-text preparation instructions. Maximum 200 characters.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="delivery_address" type="object">
  Replaces the **entire** delivery address on the order.

  <Expandable title="Address object fields">
    <ParamField body="delivery_address.street" type="string" required>
      Street address including house/apartment number.
    </ParamField>

    <ParamField body="delivery_address.city" type="string" required>
      City name.
    </ParamField>

    <ParamField body="delivery_address.state" type="string" required>
      Two-letter state abbreviation.
    </ParamField>

    <ParamField body="delivery_address.zip" type="string" required>
      ZIP or postal code.
    </ParamField>

    <ParamField body="delivery_address.country" type="string" default="US">
      ISO 3166-1 alpha-2 country code. Defaults to `US`.
    </ParamField>
  </Expandable>
</ParamField>

## Example Request

<CodeGroup>

```bash cURL
curl -X PATCH https://api.pizzaapi.com/v1/orders/order_55ABCD112233 \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "items": [
      {
        "menu_item_id": "item_01HXYZ123ABC",
        "size": "large",
        "quantity": 2,
        "toppings": ["topping_extra_cheese"],
        "special_instructions": "Extra crispy please."
      }
    ],
    "delivery_address": {
      "street": "100 New Street",
      "city": "Springfield",
      "state": "IL",
      "zip": "62701",
      "country": "US"
    }
  }'
```

```javascript JavaScript
const orderId = "order_55ABCD112233";

const response = await fetch(
  `https://api.pizzaapi.com/v1/orders/${orderId}`,
  {
    method: "PATCH",
    headers: {
      Authorization: "Bearer YOUR_API_KEY",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      items: [
        {
          menu_item_id: "item_01HXYZ123ABC",
          size: "large",
          quantity: 2,
          toppings: ["topping_extra_cheese"],
          special_instructions: "Extra crispy please.",
        },
      ],
      delivery_address: {
        street: "100 New Street",
        city: "Springfield",
        state: "IL",
        zip: "62701",
        country: "US",
      },
    }),
  }
);

const updatedOrder = await response.json();
console.log(updatedOrder);
```

</CodeGroup>

## Error Responses

| Status | Code | Description |
|---|---|---|
| `409 Conflict` | `order_not_modifiable` | The order status is no longer `pending` and cannot be modified. |
| `404 Not Found` | `order_not_found` | No order exists with the provided ID. |
| `422 Unprocessable Entity` | `validation_error` | One or more request body fields failed validation. |
