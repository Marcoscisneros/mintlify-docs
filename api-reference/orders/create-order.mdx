---

Place a new pizza order on behalf of a customer. Submit one or more line items — each specifying a menu item, size, quantity, and optional toppings — along with a delivery address. The order is created with a `pending` status and can be modified until it is confirmed by the kitchen.

```
POST https://api.pizzaapi.com/v1/orders
```

## Request Body

<ParamField body="customer_id" type="string" required>
  The unique ID of the customer placing the order. Must correspond to an existing customer record.
</ParamField>

<ParamField body="items" type="array" required>
  Array of line item objects representing the pizzas and other products being ordered. Must contain at least one item.

  <Expandable title="Item object fields">
    <ParamField body="items[].menu_item_id" type="string" required>
      The ID of the menu item to order (e.g. `item_01HXYZ123ABC`).
    </ParamField>

    <ParamField body="items[].size" type="string" required>
      Desired size for this item. Accepted values: `small`, `medium`, `large`.
    </ParamField>

    <ParamField body="items[].quantity" type="integer" required>
      Number of this item to include. Minimum: `1`.
    </ParamField>

    <ParamField body="items[].toppings" type="array">
      Array of topping IDs to add to this item. Each ID must be present in the item's `available_toppings` list.
    </ParamField>

    <ParamField body="items[].special_instructions" type="string">
      Free-text preparation instructions for this item (e.g. "extra crispy crust"). Maximum 200 characters.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="delivery_address" type="object" required>
  The address to deliver the order to.

  <Expandable title="Address object fields">
    <ParamField body="delivery_address.street" type="string" required>
      Street address including house/apartment number (e.g. `123 Main St, Apt 4`).
    </ParamField>

    <ParamField body="delivery_address.city" type="string" required>
      City name.
    </ParamField>

    <ParamField body="delivery_address.state" type="string" required>
      Two-letter state abbreviation (e.g. `CA`, `NY`).
    </ParamField>

    <ParamField body="delivery_address.zip" type="string" required>
      ZIP or postal code.
    </ParamField>

    <ParamField body="delivery_address.country" type="string" default="US">
      ISO 3166-1 alpha-2 country code. Defaults to `US`.
    </ParamField>
  </Expandable>
</ParamField>

## Response Fields

<ResponseField name="id" type="string">
  Unique identifier for the newly created order.
</ResponseField>

<ResponseField name="status" type="string">
  Current order status. Always `pending` immediately after creation. Progresses through `confirmed` → `preparing` → `out_for_delivery` → `delivered`.
</ResponseField>

<ResponseField name="customer_id" type="string">
  ID of the customer who placed the order.
</ResponseField>

<ResponseField name="items" type="array">
  The resolved line items included in this order, with pricing details applied.

  <Expandable title="Line item fields">
    <ResponseField name="items[].menu_item_id" type="string">
      ID of the menu item.
    </ResponseField>

    <ResponseField name="items[].name" type="string">
      Display name of the menu item.
    </ResponseField>

    <ResponseField name="items[].size" type="string">
      Size selected for this line item.
    </ResponseField>

    <ResponseField name="items[].quantity" type="integer">
      Quantity ordered.
    </ResponseField>

    <ResponseField name="items[].toppings" type="array">
      Array of topping IDs applied to this line item.
    </ResponseField>

    <ResponseField name="items[].special_instructions" type="string | null">
      Any special preparation instructions, or `null` if none were provided.
    </ResponseField>

    <ResponseField name="items[].unit_price" type="number">
      Price per unit for this item and size in USD.
    </ResponseField>

    <ResponseField name="items[].line_total" type="number">
      Total price for this line item (`unit_price × quantity`) in USD.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="subtotal" type="number">
  Sum of all line item totals, before tax, in USD.
</ResponseField>

<ResponseField name="tax" type="number">
  Calculated tax amount in USD.
</ResponseField>

<ResponseField name="total" type="number">
  Final order total including tax, in USD.
</ResponseField>

<ResponseField name="delivery_address" type="object">
  The delivery address on file for this order.

  <Expandable title="Address fields">
    <ResponseField name="delivery_address.street" type="string">Street address.</ResponseField>
    <ResponseField name="delivery_address.city" type="string">City name.</ResponseField>
    <ResponseField name="delivery_address.state" type="string">State abbreviation.</ResponseField>
    <ResponseField name="delivery_address.zip" type="string">ZIP or postal code.</ResponseField>
    <ResponseField name="delivery_address.country" type="string">Country code.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the order was created.
</ResponseField>

<ResponseField name="estimated_delivery_time" type="string">
  ISO 8601 timestamp of the estimated delivery window.
</ResponseField>

<Note>
  To modify line items or the delivery address after creation, use `PATCH /orders/{id}`. Changes are only permitted while the order status remains `pending`.
</Note>
